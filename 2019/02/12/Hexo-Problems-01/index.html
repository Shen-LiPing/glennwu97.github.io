什么是Hexo？

Hexo是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。Hexo官网传送门

准备材料

  • Node.js
  • Git
  • 一个编辑器Sublime/Vscode/Atom等等

PS: 由于墙的原因在下载前两者时最好挂个梯子否则可能速度极慢最后还下载失败,也可以从这个仓库下。传送门

安装

具体安装过程官网写的非常详尽了,故在此不再复述.Hexo官网安装传送门

我的安装流程

  1. 买个我喜欢的域名
  2. 下载Node.js、Git
  3. 利用Git安装Hexo，并进行本地测试
  4. 注册github账号,创建一个myname.github.io的仓库
  5. 部署本地文件到Github进行查看
  6. Github Pages绑定域名
  7. 安装Next主题,并进行优化,然后换了一个主题…
  8. 优化完成开始码字写博客

PS:当时以为非.cn就能备案就买了两个😵,结果是一样的(直接被墙😭),除非去境外域名商如GoDaddy购买域名。

这里要进行注解的是，有的教程会让配置SSH密钥，SSH密钥可以为了防止其他人恶意部署文件到你的仓库，使用以后每次部署文件都需要登录Github账号，可以不配置。
有的教程中还会要求下载Github客户端，但我觉得没什么卵用。

遇到的坑

YAML 解析错误

 S-YAML: incomplete explicit mapping pair; a key node is missed at line 18, 
 column 29: last_updated: Last updated: %s

如果 YAML 字符串中包含冒号(:)的话，请加上引号。

JS-YAML: bad indentation of a mapping entry at line 18, column 31:
      last_updated:"Last updated: %s"

确认使用空格进行缩进（Soft tab），并确认冒号后有加上一个空格

Git 部署问题

fatal: 'username.github.io' does not appear to be a git repository

请确认已经在电脑上 配置 git，或改用 HTTPS 库（repository）地址。

找不到git部署

ERROR Deployer not found: git

在Git Bash命令行中输入: npm install hexo-deployer-git –save

服务器问题

Error: listen EADDRINUSE
您可能同时开启两个 Hexo 服务器，或者有其他应用程序正在占用相同的端口，请尝试修改 port 参数，或是在启动 Hexo 服务器时加上 -p 选项。

$ hexo server -p 5000

更新完文档并部署后主页无变化

有时资料可能没有被更新，或是生成的文件与修改前的相同,可以尝试清除缓存并再执行一次。

$ hexo clean

.md文件错误


这个错误就是在提示我md文件的tags标签处出现了错误，tags：后没加空格,注意.md以及.yml文件中的:后都要加上一个空格,否则很可能在渲染时就会报出上述错误。

一点心得

Git是啥?

Git是Git Bash不是Github更不是Github Shell，是一个命令行工具，长这个样子。

Node.js又是啥?

Node 是一个让 JavaScript 运行在服务端的开发平台，它让 JavaScript 成为与PHP、Python、Perl、Ruby 等服务端语言平起平坐的脚本语言。简单的说它是一个语言转化平台,在这里这要将其下载到文件内并不需要了解过多。

关于Hexo 所带的文件结构

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes
  • 每个Hexo主题文件(theme文件)中有.yml脚本文件，通常称其为主题配置文件;在根目录下也有个.yml文件称为站点配置文件
  • 优化主题时，修改Hexo根目录下的_config.yml站点配置目录后，要重新启动Hexo再刷新，修改next主题目录下的_config.yml主题配置目录则只需刷新即可.

Hexo本地测试的时候要注意

  1. Git要运行在本地Hexo的根目录下。
  2. Git中Ctrl + C是stop的快捷键，不要使用Ctrl + C或Ctrl + V的快捷键，用右键。
  3. 本地测试是Hexo is running,Git显示如下，其他状态便无法连接
    $ hexo s
INFO  Start processing
INFO  Hexo is running at https://localhost:4000/. Press Ctrl+C to stop.

关于使用Github Pages进行代码托管

  • Github Page提供的300M免费空间可以作为服务器
  • Github page托管Hexo代码时要注意一个github用户名只能有一个io,一个io也只能映射一个域名,一经绑定好域名后再次更换映射的域名可能出现输入相应的io跳转到老域名的情况,反正在我这是这样的，所以绑定io仓库的域名需谨慎。
  • 绑定Github Page仓库的名称格式为 Github.yourname.io 不识别大小写但是千万不可打错单词。
  • Github Page建立好后会提供一个IP,获得这个IP的方法就是在CMD中Ping一下建立好的Github Page仓库地址。即yourname.github.io

解析与绑定域名

在用相应云解析平台提供的dns解析域名时解析一级域名(www.xxx.xx)时需要添加两条记录。一条`www`记录 一条@记录 添加完解析记录后 将解析完成后的域名如gerlan.cn写入并保存在一个CNAME文件放在Hexo主题文件夹的source文件夹下即可完成域名关联。


注意:

  • 在部署前放在根目录中直接使用Git Bash部署到io仓库后不会完成域名关联。
  • CNAME是一个没有任何后缀的文件,千万不要加上.txt一类的后缀。
  • 在填写解析线路时使用默认路线,别使用境外路线，否则你的网页可能还没开始使用就被墙了。
  • 若使用二级域名则在解析时只需要添加一条记录 xx(即你的二级域名名称)

最后:
貌似有说法是使用Coding Pages 托管网页代码会比Github Pages 快一些,但是Coding的云主机地址也是在海外的所以我觉得应该也差不多。
