Visual Studio Code折腾记:JavaSE开发环境搭建

Glenn 3月 05, 2019
  • Visual Studio Code
  • 分享到微博
  • 分享到 Facebook
  • 分享到 QQ

效果:

在工作区新建一个.java文件在直接在Vscode编辑器内运行测试代码。

准备材料:

1.Visual Studio Code
2.jdk-8u201-windows-x64.exe
3.拓展-Code Runner

系统: Windows-10-x64

背景知识:

1.首先要清楚Visual Studio Code 本身只是一个文本文本编辑器只是用来编辑代码的,若想用它来编译,调试和运行代码则需要在它提供的拓展库中装相应的拓展插件并在外部配置好执行代码所需要编译器,解释器的路径(环境变量)。

2.Java是编译与解释两者之间的混合式语言。Java源程序必须经过编译才能运行。编译器是一种将程序源代码转换成可执行格式(如字节码)的程序，Java的编译器是一个名为javac的程序。使用javac可将Java源代码编译成字节码,运行字节码则需要一个Java虚拟机(JVM)。此外,开发中常用到Java核心类库中的类,因此还需要下载这些类库。JVM和Java类库一起构成了Java运行时环境(Java runtime enviroment,JRE)。 PS：不同操作系统的JRE并不相同。

3.JDK(Java Development Toolkit): 称为Java开发工具包。包括JRE，外加一个编译器和其他工具，是编译和运行Java环境必备的软件。 简而言之,JVM是一种运行字节码的应用程序。JRE则是一种包含了JVM和Java类库的环境。JDK则包含JRE及一个Java编译器和其他程序的工具集。

配置步骤:

1.下载jdk

jdk下载完成后打开后一路选择(下一步)next即可,如果不想装在C盘就设置下安装路径选择自己要安装的路径。PS:记住自己安装的路径，配置环境的时候会用到

2.配置环境变量
安装好了之后,打开此电脑->系统属性->查找编辑系统环境变量->环境变量->系统变量栏下的新建->需要配置三个环境


-变量名：JAVA_HOME
-变量值：C:\Program Files\Java\jdk1.8.0_201 （即你JDK的安装路径）

-变量名：CLASSPATH
-变量值：.;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar （留意前面有个小点 “.”）

-变量名：PATH
-变量值：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;（在变量值最后输入注意原来Path的变量值末尾有没有;号，如果没有，先输入；号再输入上面的代码）
最后点击确定确定，环境就配置完成了。

1.下载Visual studio code扩展库中的 Code Runner

使用cmd检查配置是否安装成功:

打开cmd回车->输入java-> 输入javac -> 输入java -version ->若均能显示相关信息则代表安装完成。


使用Visual Studio Code运行测试代码:

点击了右上角的右向三角按钮就可以在自带的终端下运行代码。然后下方的终端里就会出现一个炫酷无敌diao炸天的Hello World,也就代表一个基于CodeRunner的Java学习开发环境已经搭建完成啦!

常见的坑:

1.编辑系统环境变量的时候注意系统变量CLASSPATH路径配置的时候要在首位加上,;否则就会出现在cmd中用javac编译可以通过但用java命令后却出现无法发现类这一情况。

2.在java源文件的开头不要加上package xx, java源文件开头的package xx是Eclipse等开发Java大型项目的IDE所特有的格式,javac编译器可以通过编译但是java解释器是无法识别该格式的。所以在用不使用IDE开发的情况下源文件开头就不要加上这句话了。
